# Agent Documentation: appkit-wagmi

This file provides guidance to AI agents when working with code in this repository.

## Project Overview

**appkit-wagmi** is a React Native sample application demonstrating the **Reown AppKit SDK** for multichain Web3 integration. It showcases wallet connection, signing, and transaction capabilities across multiple blockchain ecosystems:

- **EVM chains** (Ethereum, Polygon, Arbitrum, Optimism, Base, etc.)
- **Solana** (via Phantom & Solflare wallets)
- **Bitcoin** (PSBT signing)

This is a reference implementation for developers building dApps with Reown AppKit.

## Tech Stack

- **React Native**: 0.85.3

### Key Libraries

## Environment Variables

Required in `.env` (Expo inlines `EXPO_PUBLIC_*` vars into the JS bundle at build time):
```bash

EXPO_PUBLIC_SENTRY_DSN=""        # Sentry error tracking (optional)
SENTRY_DISABLE_AUTO_UPLOAD=true  # Build-time only: disable Sentry auto upload
```

## Development

### Setup
```bash
yarn install
cp .env.example .env
# Edit .env with your project ID
yarn android    # or yarn ios
```

### Scripts
- `yarn start`: Start Metro bundler
- `yarn prebuild`: Generate native `ios/` + `android/` from config (runs automatically before `run`)
- `yarn android`: Prebuild + run on Android (debug)
- `yarn ios`: Prebuild + run on iOS
- `yarn ios:internal` / `yarn android:internal`: Run the internal variant
- `yarn android:build` / `yarn android:build:internal`: Build release / internal APK
- `yarn lint`: Run ESLint
- `yarn test`: Run Jest tests

### Build Variants

Selected via the `APP_VARIANT` env var (iOS, via `app.config.js`) and Gradle
buildTypes (Android, via `plugins/withAndroidVariants.js`):

- **debug** — local development (`…rnsample.debug`, scheme `w3mwagmisample-debug://`)
- **internal** — internal distribution (`…rnsample.internal`, scheme `w3mwagmisample-internal://`)
- **production** — release (`…rnsample`, scheme `w3mwagmisample://`)

```typescript
import { WagmiProvider } from 'wagmi';

// Initialize AppKit with project ID and chains

  projectId: process.env.EXPO_PUBLIC_PROJECT_ID,
  // ... configuration
```


## Code Quality Guidelines

**Always run these checks and fix any errors before committing:**

```bash
yarn lint             # Check and fix ESLint errors
yarn prettier --write . # Format code with Prettier
npx tsc --noEmit      # Check for TypeScript errors
```

### Style
- Use TypeScript strict mode
- Follow ESLint + Prettier configuration
- Use path aliases (`@/`) for imports
- Handle errors with toast notifications
- Log important events for debugging
